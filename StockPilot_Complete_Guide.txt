1.	Project	Overview	&	What	You're	Building
StockPilot
	is	an	internal	warehouse	operations	tool	—	the	kind	of	software	a	company	like	Amazon	or	Flipkart	runs	behind	the
scenes	to	track	what	stock	sits	in	which	warehouse,	reserve	items	while	a	customer	is	paying,	move	stock	between	locations,
and	keep	an	audit	trail	of	who	changed	what.	It	is	
not
	a	customer-facing	storefront;	there	is	no	shopping	cart	or	checkout	for
the	public.	Staff	and	admins	log	in,	manage	inventory,	and	process	orders.
The	application	is	built	as	two	connected	halves:
Backend
	—	a	Spring	Boot	REST	API	that	holds	all	business	rules	(reservations,	transfers,	locking)	and	talks	to	MySQL.
Frontend
	—	a	React	single-page	app	that	logs	in,	shows	dashboards	and	tables,	and	calls	the	backend	over	HTTP.
The	whole	thing	runs	with	a	single	
docker	compose	up
,	which	is	exactly	the	"it	just	works"	impression	you	want	when
demonstrating	it	in	an	interview.
The	features	that	make	this	stand	out
Most	student	inventory	projects	stop	at	"product	has	a	stock	number,	subtract	when	you	buy."	StockPilot	goes	further,	and
these	three	pieces	are	what	you	should	talk	about	in	interviews:
Feature

Reservation
system
Stock	is	
reserved
	during	checkout,	not	immediately	deducted.	Available	=	OnHand	−	Reserved.	This	mirrors	how

10.	Step	4	—	Core	Business	Logic

10.1	The	reservation	flow
Stock	=	10
			│		reserve	2
			
▼
onHand=10,	reserved=2		→		available	=	8
			│		payment	success	→	confirm
			
▼
onHand=8,		reserved=0		→		available	=	8
			(or)	payment	never	comes	→	expiry	job	releases	→	reserved	back	to	0
ReservationService.java
--- PAGE 11 ---
@Service
@RequiredArgsConstructor
public	class	ReservationService	{
				private	final	InventoryRepository	inventoryRepo;
				private	final	ReservationRepository	reservationRepo;
				private	final	AuditLogService	auditLog;
				@Transactional
				public	InventoryReservation	reserve(ReserveStockRequest	req)	{
								Inventory	inv	=	inventoryRepo
												.findByProductIdAndWarehouseId(req.productId(),	req.warehouseId())
												.orElseThrow(()	->	new	NotFoundException("Inventory	not	found"));
								if	(inv.getAvailable()	<	req.quantity())	{
												throw	new	InsufficientStockException(
																"Only	"	+ 	inv.getAvailable()	+ 	"	available");
								}
								inv.setQuantityReserved(inv.getQuantityReserved()	+ 	req.quantity());
								inventoryRepo.save(inv);			//	@Version	guards	against	lost	updates
								InventoryReservation	r	=	InventoryReservation.builder()
												.inventory(inv)
												.quantity(req.quantity())
												.status(ReservationStatus.ACTIVE)
												.expiresAt(LocalDateTime.now().plusMinutes(15))
												.salesOrderId(req.salesOrderId())
												.build();
								auditLog.record("RESERVE",	"Inventory",	inv.getId(),
																								"Reserved	"	+ 	req.quantity());
								return	reservationRepo.save(r);
				}
				@Transactional
				public	void	confirm(Long	reservationId)	{
								InventoryReservation	r	=	reservationRepo.findById(reservationId)
												.orElseThrow(()	->	new	NotFoundException("Reservation	not	found"));
								Inventory	inv	=	r.getInventory();
								//	Reserved	stock	actually	leaves	the	warehouse
								inv.setQuantityReserved(inv.getQuantityReserved()	-	r.getQuantity());
								inv.setQuantityOnHand(inv.getQuantityOnHand()	-	r.getQuantity());
								r.setStatus(ReservationStatus.CONFIRMED);
								inventoryRepo.save(inv);
								reservationRepo.save(r);
								auditLog.record("CONFIRM",	"Inventory",	inv.getId(),
																								"Confirmed	"	+ 	r.getQuantity());
				}
				@Transactional
				public	void	release(InventoryReservation	r)	{
								Inventory	inv	=	r.getInventory();
								inv.setQuantityReserved(inv.getQuantityReserved()	-	r.getQuantity());
								r.setStatus(ReservationStatus.RELEASED);
								inventoryRepo.save(inv);
								reservationRepo.save(r);
				}
}
ReservationExpiryJob.java	—	scheduled	release
--- PAGE 12 ---
@Component
@RequiredArgsConstructor
public	class	ReservationExpiryJob	{
				private	final	ReservationRepository	reservationRepo;
				private	final	ReservationService	reservationService;
				private	final	NotificationService	notificationService;
				@Scheduled(fixedRate	=	60_000)			//	every	minute
				@Transactional
				public	void	expireStaleReservations()	{
								var	expired	=	reservationRepo.findByStatusAndExpiresAtBefore(
												ReservationStatus.ACTIVE,	LocalDateTime.now());
								for	(InventoryReservation	r	:	expired)	{
												reservationService.release(r);
												r.setStatus(ReservationStatus.EXPIRED);
												notificationService.push(NotificationType.RESERVATION_EXPIRED,
																"Reservation	"	+ 	r.getId()	+ 	"	expired");
								}
				}
}

@EnableScheduling

10.2	The	atomic	stock	transfer
This	is	probably	your	strongest	module.	Moving	stock	from	one	warehouse	to	another	must	be	all-or-nothing:	if	the	third	item
in	a	five-item	transfer	fails,	the	first	two	must	roll	back.	A	single	
@Transactional
	method	gives	you	exactly	that	—	any
exception	triggers	a	full	rollback.
StockTransferService.java
@Service
@RequiredArgsConstructor
public	class	StockTransferService	{
				private	final	InventoryRepository	inventoryRepo;
				private	final	StockTransferRepository	transferRepo;
				private	final	NotificationService	notificationService;
				private	final	AuditLogService	auditLog;
				@Transactional			//	all	items	move,	or	none	do
				public	StockTransfer	transfer(StockTransfer	transfer)	{
								for	(StockTransferItem	item	:	transfer.getItems())	{
												Inventory	from	=	inventoryRepo.findByProductIdAndWarehouseId(
																item.getProductId(),	transfer.getFromWarehouse().getId())
																.orElseThrow(()	->	new	NotFoundException("Source	inventory"));
												if	(from.getAvailable()	<	item.getQuantity())	{
																throw	new	InsufficientStockException(
																				"Not	enough	stock	to	transfer	product	"
																				+ 	item.getProductId());			//	→	rolls	back	everything
												}
												from.setQuantityOnHand(from.getQuantityOnHand()	-	item.getQuantity());
												Inventory	to	=	inventoryRepo.findByProductIdAndWarehouseId(
																item.getProductId(),	transfer.getToWarehouse().getId())
																.orElseGet(()	->	createEmptyInventory(
																				item.getProductId(),	transfer.getToWarehouse()));
												to.setQuantityOnHand(to.getQuantityOnHand()	+ 	item.getQuantity());
												inventoryRepo.save(from);
												inventoryRepo.save(to);
								}
								transfer.setStatus(TransferStatus.COMPLETED);
								StockTransfer	saved	=	transferRepo.save(transfer);
								auditLog.record("TRANSFER",	"StockTransfer",	saved.getId(),
												"Transfer	"	+ 	transfer.getFromWarehouse().getCode()
												+ 	"	→	"	+ 	transfer.getToWarehouse().getCode());
								notificationService.push(NotificationType.TRANSFER_COMPLETED,
												"Transfer	#"	+ 	saved.getId()	+ 	"	completed");
								return	saved;
				}
}

18.	Interview	Talking	Points
Rehearse	crisp	answers	to	these	—	they're	what	turn	a	project	on	your	resume	into	a	conversation	you	win.
Likely	question
Your	answer,	in	one	breath
"Why	reserve	instead	of	just
subtracting?"
To	avoid	overselling	during	checkout.	Stock	is	held	(reserved),	not	removed,	until	payment
confirms	—	and	released	automatically	if	it	doesn't.
"What	if	two	people	edit	the
same	inventory	at	once?"
Optimistic	locking	via	
@Version
.	The	second	save	fails	with	an	optimistic-lock	exception	instead
of	silently	overwriting;	I	return	a	409	asking	the	client	to	retry.
"How	is	a	multi-item	transfer
safe?"
The	whole	transfer	runs	in	one	
@Transactional
	method.	Any	shortfall	throws,	Spring	rolls	back,
and	no	partial	move	persists.
"How	do	expired	reservations
get	cleaned	up?"
A	
@Scheduled
	job	runs	every	minute,	finds	ACTIVE	reservations	past	their	expiry,	releases	the
reserved	quantity,	and	marks	them	EXPIRED.
"Why	DTOs	instead	of	returning
entities?"
To	decouple	the	API	from	the	schema,	hide	internal	fields	like	the	version	column,	and	avoid	lazy-
loading	serialization	issues.
"How	do	you	secure	admin
actions?"
Stateless	JWT	auth	plus	method-level	
@PreAuthorize("hasRole('ADMIN')")
	on	transfer	and
delete	endpoints.
